# FunPay Automation Tool

Браузерное расширение Manifest V3 для автоматизации рутинных действий продавца
на FunPay.

## Возможности

- ручное и автоматическое поднятие активных категорий каждые 4 часа;
- мультипостинг из формы создания или редактирования объявления;
- поиск по понятным именам вида `Gemini · Услуги` и `Claude · Токены`;
- адаптация данных к форме каждой целевой категории вместо слепой замены `node_id`;
- индивидуальные черновики копий: тексты RU/EN, описание, сообщение покупателю,
  цена, списки и переключатели можно изменить отдельно для каждой категории;
- последовательная отправка запросов с прогрессом и результатом по каждой категории;
- локальная история последних операций.

### Как адаптируется мультипостинг

Перед созданием каждой копии расширение загружает актуальную форму выбранной
категории и начинает запрос с ее скрытых полей и значений по умолчанию. Затем оно
переносит только совместимые данные исходного объявления: краткое и подробное
описание, сообщение покупателю, цену, изображения и состояние активности.

Поля, которых нет в целевой форме, не отправляются. Например, при переносе из
`ChatGPT · Прочее` в `Gemini · Услуги` параметры автовыдачи и наличия будут
отброшены. Для разделов `Услуги` опция «Деактивировать после продажи» принудительно
остается выключенной, пока пользователь явно не изменит ее в черновике категории.

После выбора категории нажмите на ее название в блоке «Будут созданы копии».
Расширение загрузит настоящую форму целевой категории и откроет редактор уже
заполненной копии. Сохраненные переопределения отмечаются статусом «Настроено» и
не изменяют исходное объявление или другие выбранные категории.

## Установка для разработки

1. Откройте `chrome://extensions`.
2. Включите режим разработчика.
3. Нажмите **Загрузить распакованное расширение**.
4. Выберите папку проекта.

После изменения исходников нажмите кнопку обновления на карточке расширения.

## Важные замечания

- Расширение использует текущую браузерную сессию FunPay и не хранит пароль.
- Автоматизация не гарантирует отсутствие ограничений со стороны площадки.
- Мультипостинг зависит от HTML-форм и формата ответа FunPay. После изменений сайта
  совместимость нужно проверять вручную.
- Перед массовой публикацией сначала проверьте одну дополнительную категорию.

## Структура

- `background.js` — тонкий entrypoint service worker и маршрутизация сообщений;
- `background/` — клиент FunPay, сервисы, парсеры и классификация результатов;
- `content.js`, `content/` и `content.css` — entrypoint, Controller/View/Client и
  стили панели мультипостинга;
- `popup.html` / `popup.css` / `popup.js` — управление расширением и история;
- `tests/` — unit-тесты парсеров, фоновых сервисов и адаптера форм;
- `manifest.json` — конфигурация Manifest V3.

## Проверка

Проект не требует сборки или runtime-зависимостей. Для локальной проверки нужен
Node.js:

```powershell
npm run verify
```

Команда проверяет синтаксис всех JavaScript-файлов, ссылки из manifest и запускает
unit-тесты. Тот же набор автоматически выполняется в GitHub Actions для `main` и
pull request.

## Архитектурные решения

- Service worker использует ES-модули и разделяет транспорт, parsing и бизнес-логику.
- `BumpService` и каталог категорий получают зависимости через конструктор, поэтому
  тестируются без Chrome и реальных запросов к FunPay.
- Content scripts загружаются в одном изолированном Chrome-контексте и разделены на
  `Controller`, `View`, HTTP-клиент и адаптер целевых форм через единый namespace
  расширения.
- Service worker загружает каталог через MAIN-мир открытой вкладки FunPay, чтобы
  запрос использовал тот же origin, cookies и сетевой контекст, что и сам сайт.
- Небольшие helpers сетевого ответа намеренно продублированы между service worker и
  content script: эти части работают в разных контекстах Chrome, а общий runtime-модуль
  потребовал бы сборщика или глобального namespace.
